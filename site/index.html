sender-trades

LLM-powered intraday directional prediction engine for SPY and QQQ. Ingest the Atlas Morning Briefing, run LLM research, and get a clear % direction prediction with cited evidence — all on your own infrastructure.

● 127 tests ● LLM-driven predictions ● Per-asset % forecasts ● Snapshot-aware ● Prediction history & learning

Features

🤖

LLM-powered prediction

Asks the opencode CLI to emit per-asset directional predictions (direction, confidence %, expected move %, rationale, sources). Free Zen models first, paid Go fallback. Traces predictions back to original news publishers and market data — not the briefing itself.

📊

Clear forecast table

No confusing UP/DOWN/SIDE columns. Each asset gets a simple readout: Direction (UP/DOWN), Confidence, Predicted Move %, and Key Drivers of the Prediction with cited evidence. Also includes a Market Vibe summary for overall context.

🔍

Root-provenance citation

The LLM must cite where each prediction's evidence came from — original publisher (reuters, bloomberg, seekingalpha), watchlist ticker, or market data. No more opaque "LLM said so" or "per the briefing."

📈

Momentum + MeanReversion + EventDriven

Three deterministic strategies run alongside the LLM for cross-reference: gap-based momentum, RSI-based mean-reversion, and catalyst-aware event-driven logic from macro data and news polarity.

🎯

Realistic 0DTE strikes

Strike computation fixed — no more 15% OTM strikes that don't exist in the chain. 30-delta options now target ~0.6% OTM, producing strikes like QQQ PUT @ 691 (not 591) for a 695 underlying.

📰

Multi-source ingestion

Finnhub quotes, Brave news, RSS feeds, and Atlas morning briefings. Snapshot-aware — reuses cached data from atlas-morning-briefing to avoid duplicate API calls and rate limits.

📋

Structured logging

JSON logs with per-run correlation IDs. Every prediction, risk check, and execution step is traceable. Logs organized by date in logs/YYYY-MM-DD/ with run and summary files.

📬

Email reports

Styled HTML forecast sent via Gmail SMTP. Clear table with Direction, Confidence, Predicted Move, and Key Drivers of the Prediction per asset. Each send is persisted to logs/<date>/email-<correlation_id>.html for audit.

🔄

Yesterday's prediction recap

After each run, checks the previous business day's prediction against finhub 1-hour candles. Determines if the predicted direction came true at any point intraday, when it triggered, and how long it held. Success/fail results appear in the email and are persisted to logs/prediction-history.json.

🧠

Learning from history

Prediction history is pruned and injected into the LLM's user prompt — prioritizing recent misses — so the model can learn from past outcomes. Aggregate success rate and per-prediction rationales help refine future calls without unbounded prompt growth.

🕐

LA timezone aware

All dates and times use America/Los_Angeles — log directories, email timestamps, briefing lookup, and risk engine deadlines. Market-close checks use actual America/New_York time.

🛡️

Risk guardrails (optional)

When best_trade execution is enabled: max loss cap, realistic 0DTE strikes, bid/ask spread ≤20%, time checks (no entries after 2PM), and OCC chain existence validation.

🐳

Docker + CI/CD

Multi-stage Dockerfile, docker-compose for app + docs, GitHub Actions for CI (ruff + pytest, 127 tests) and GitHub Pages deploy.

Snapshot flow

atlas-morning-briefing saves raw API responses to snapshots/YYYY-MM-DD/. sender-trades checks this directory before making any API call. If snapshot has quotes for all target assets plus news — zero API calls are made. This avoids duplicate Finnhub/Brave charges and rate limits.

End-to-end example

A live prediction run from 2026-07-18 against the atlas-morning-briefing snapshot. Three stages: ingest briefing + market → LLM predicts per-asset → email delivered. Correlation ID: test-run-001.

1 Ingest — Atlas briefing + market snapshots atlas-morning-briefing repo →

Parsed briefing from ~/atlas-morning-briefing/briefings/Atlas-Briefing-2026.07.18.md. Upstream runs an LLM distillation pass and ships a 25-row watchlist, executive summary, AI & tech news, blog updates, and top papers.

Market data loaded from snapshot snapshots/2026-07-18/ — 241 news items, 24 quotes, 67 RSS entries. Zero API calls made.

2 LLM prediction — per-asset forecast opencode/deepseek-v4-flash-free · 17.2s · free Zen tier

LLM prompt asked for predictions covering ALL target assets (SPY and QQQ), not just a single trade. The free Zen model opencode/deepseek-v4-flash-free responded first.

The LLM was instructed to cite root provenance — tracing evidence back to the original news publisher (reuters, bloomberg, etc.) or market data — not the atlas briefing itself. The response included both per-asset predictions and an optional best_trade.

Quick start

git clone <repo> && cd sender-trades
Clone the repository
cp .env.example .env
Configure API keys (Finnhub, Brave, Alpaca)
uv sync --all-extras
Install dependencies (uv recommended)
uv run python -m src.main --dry-run
Run a dry run — no trades executed
uv run python -m src.main --dry-run --email
Dry run + send forecast email
docker compose run --rm app python -m src.main --dry-run
Or run via Docker
uv run pytest
Run all 127 tests
uv run ruff check && uv run ruff format --check
Lint and format check
