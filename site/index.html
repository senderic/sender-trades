sender-trades

0DTE intraday options trading recommendation system for SPY and QQQ. Ingest, analyse, decide, execute — on your own infrastructure. Snapshot-aware: reuses atlas-morning-briefing data to avoid duplicate API calls.

● 126 tests ● 4 strategies (LLM-driven) ● 8 risk guardrails ● Snapshot-aware

Features

📈

Momentum strategy

Detects gap openings and intraday momentum breaks on SPY and QQQ. Triggers on opening range breakouts and trendline violations with volume confirmation.

↔️

Mean-reversion strategy

RSI-based contrarian signals when the market over-extends. Enters fade positions when RSI crosses below 35 (oversold) or above 65 (overbought) on 5-minute bars.

⚡

Event-driven strategy

Catalyst-aware recommendations from news sentiment and macro data. Parses Atlas morning briefings for macro sentiment, sector rotation, and thematic shifts.

🤖

LLM trade-signal strategy

Asks the opencode CLI to emit a structured JSON pick (asset / direction / confidence / rationale / sources) over the briefing + market snapshot. Free Zen first, paid Go fallback (60s timeout). Surfaces root provenance — e.g. ↓llm:atlas-briefing:executive_summary, llm:watchlist:NVDA — in the forecast table.

🧩

Decision aggregator

Picks the highest-confidence recommendation or merges near-ties with 2-of-3 consensus. Produces a directional forecast table (UP / DOWN / SIDEWAYS confidence + expected MOVE %).

🛡️

8 risk guardrails

Max loss cap (0.3% underlying premium estimate), bid/ask spread ≤20%, statistical sanity (≤2% move), time checks (no trades after 2PM, close by 3:30PM), OCC chain existence, 2-of-3 consensus, rationale schema.

🔌

MCP broker abstraction

Subprocess stdio transport for Alpaca and Robinhood. Dry-run by default — nothing leaves your machine. Use --execute to enable live trading.

📰

Multi-source ingestion

Finnhub quotes, Brave news, Reddit WSB/options sentiment, Unusual Whales options flow, RSS feeds, and Atlas morning briefings. Snapshot-aware — reuses cached data from atlas-morning-briefing to avoid duplicate API calls and rate limits.

📋

Structured logging

JSON logs with per-run correlation IDs. Every decision, risk check, and execution step is traceable. Logs organized by date in logs/YYYY-MM-DD/ with run and summary files.

📬

Email reports

Styled HTML forecast table sent via Gmail SMTP (app password). Each send is also persisted to logs/<date>/email-<correlation_id>.html for audit. Use --email to receive the directional forecast in your inbox each morning.

🐳

Docker + CI/CD

Multi-stage Dockerfile, docker-compose for app + docs, GitHub Actions for CI (ruff + pytest) and GitHub Pages deploy for the brochure site.

Snapshot flow

atlas-morning-briefing saves raw API responses to snapshots/YYYY-MM-DD/. sender-trades checks this directory before making any API call. If snapshot has quotes for all target assets plus news — zero API calls are made. This avoids duplicate Finnhub/Brave charges and rate limits.

2 LLM trade signal — decision opencode/deepseek-v4-flash-free · 13.1s · free Zen tier

Strategy: llm_trade — fed the briefing + market snapshot + news sentiment to the opencode CLI and asked for a single JSON pick. The free Zen model opencode/deepseek-v4-flash-free responded first (no paid Go tier needed — paid_used = false).

The LLM was asked to cite root provenance — which upstream input actually drove the call — not just say "LLM said so". It cited three sources: the briefing's executive summary, and two watchlist tickers (NVDA, META).

{ "asset": "QQQ", "direction": "PUT", "confidence": 0.62, "rationale": "Consumer-AI tech names that dominate QQQ are being sold off (NVDA -2.21%, META -2.79%, GOOGL -2.17%, MSFT -1.81%) while defense/space infrastructure holds or rallies — the agentic-defense rotation narrative from the morning briefing indicates continued capital rotation out of QQQ's heaviest weights.", "sources": [ "atlas-briefing:executive_summary", "watchlist:NVDA", "watchlist:META" ] }

Decision aggregator: selected llm_trade @ 0.62 (other three strategies abstained below threshold). Risk engine: _check_statistical_sanity flagged the 15% OTM strike and halved confidence to 0.31. Final recommendation: QQQ PUT @ $591, 1 contract, 0.31 confidence.

3 Email output — as delivered Gmail SMTP · 2/2 sent

The pipeline's forecast table is rendered into an HTML email via Gmail SMTP. The LLM's root provenance flows verbatim into the Sources column, so each email immediately shows why the LLM picked what it picked — not just the strategy name.

Risk guardrails

  • Source-verified prices — quote must come from Finnhub or snapshot
  • Chain existence — OCC option chain must be live
  • Bid/ask spread ≤20% of mid-price
  • 2-of-3 consensus — briefing sentiment, news polarity, and macro must agree
  • Statistical sanity — expected move ≤2% of underlying
  • Max loss hard cap — 0.3% of underlying (realistic 0DTE ATM premium estimate, default $1000)
  • Time check — no entries after 2PM ET, all positions closed by 3:30PM ET
  • Rationale schema — every trade must have a documented reason
